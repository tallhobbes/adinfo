D3 Interface for Reddit.com

Description
This is part of a current personal project of mine that was sparked by the desire to see how the front page of reddit evolves over time. I'm scraping and accruing front-page data into a database, which can then be rendered in an interactive timeline format. My goal is to make a publicly available live version, but what you see here is just a demo retrieving a static .json file.

It's been fun to explore mixing interaction with analysis - for instance the color tiles re-arrange to reflect the "score" for that subreddit in the displayed time range, and by isolating just a few subreddits you can see how different categories are popular at different times of day (though not with this short dataset).

I find that teaching myself things is most effective with a goal in mind, so this project has been a great platform both for getting practice with d3 and with solving emergent challenges using javascript!

MotivationSurvey.com
Description
MotivationSurvey.com was a custom-built survey administration web application that allowed a small research team to collect a large amount of data in support of an NSF-backed motivation research grant. The software was designed to reduce or eliminate many of the difficulties encountered in manual administration of precisely scheduled longitudinal surveys.

Because of our unique use-case of administering surveys each week at times specific to each class, there was no readily available off-the-shelf survey system that automated this process. Additionally we wanted to implement a smart reminder system that emailed only non-responders, which meaningfully aided our completion rates without annoying people who already participated.

The system was very successful, collecting over 5,000 responses during the three years of the project, allowing us to accrue the largest dataset of its kind while having much lower incompletion rates and administrative loads than would have occurred with a manually administered, non-customized approach. Although the site is no longer available because the research project has ended, you can still see the welcome page via the wayback machine.

This project actually launched my web app development trajectory: I used an Agile approach to coordinate with two students over the summer to build the site, but they lost momentum and wound up leaving significant portions uncompleted and non-functional. With our large-scale data collection project looming, I taught myself enough about javascript, node, jquery, and css to complete the project and add new functionality down the line. I was happy to have upheld the project goals and deliverables, and am glad this launched me into the world of modern full stack web development!

Mobile-Friendly Survey Interface using Angular.js
Description
Reviewing the Google Analytics for our motivation survey site revealed that many students were accessing the page via their mobile device. This created usability issues in the survey, which was laid out as a matrix of radio buttons in a fixed-width, non-responsive format that would be significantly obscured in a mobile browser.

To address this, I built a prototype for a responsive and more touch-friendly interface using Twitter Bootstrap, which was already used in the site, but also using the power of data-binding provided by Angular.js. This made coordinating the states of various elements on the page much cleaner than in raw javascript or JQuery, and also opened the possibility for efficient creation of new surveys by dynamically generating forms from a stored data structure.

This interface was also an improvement for desktop users: the original layout had the the meaning ascribed to the numeric responses displayed only in the column headers, which made them difficult to see as the user worked down the form. The new version made these meanings visible for each question, reducing the possibility of accidental misinterpretation and increasing accuracy.
